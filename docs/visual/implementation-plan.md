# 「念头」小程序实现方案总结

基于需求方案、视觉参考和素材分析，我整理了具体的实现方案和需要你处理的素材。

## 1. 实现方案概述

### 1.1 技术架构

```
视觉稿 → 设计系统 → 组件库 → 页面实现 → 样式优化
```

### 1.2 实现路径

**阶段1：基础搭建（1-2天）**
- 建立设计系统（颜色、字体、间距）
- 实现基础组件（卡片、按钮、输入框）
- 搭建页面框架

**阶段2：样式实现（2-3天）**
- 用CSS实现背景效果
- 用CSS实现基础图标
- 实现卡片样式

**阶段3：素材替换（2-3天）**
- 替换背景图片
- 替换图标
- 添加装饰元素

**阶段4：细节优化（1-2天）**
- 动画效果
- 适配优化
- 性能优化

## 2. 需要你处理的素材

### 2.1 P0素材（必须，第一阶段）

#### 底部导航图标（8个）
**需求描述：**
- 风格：简洁、圆润、线面结合
- 尺寸：48x48px（2倍图，实际显示24x24px）
- 格式：PNG，透明背景
- 颜色：默认状态用灰色，选中状态用暖黄色

**具体图标：**
1. **记录图标**：叶子或笔的形状
   - `tab-record.png`：轮廓版本
   - `tab-record-active.png`：填充版本，暖黄色

2. **心念本图标**：书本或纸页形状
   - `tab-book.png`：轮廓版本
   - `tab-book-active.png`：填充版本，暖黄色

3. **发现图标**：星星或罗盘形状
   - `tab-discover.png`：轮廓版本
   - `tab-discover-active.png`：填充版本，暖黄色

4. **我的图标**：人形或小屋形状
   - `tab-mine.png`：轮廓版本
   - `tab-mine-active.png`：填充版本，暖黄色

- 尺寸：750x1334px（2倍图）

**参考视觉稿：**

## 3. 开发者实现方案

### 3.1 用CSS实现的替代方案

**在素材准备好之前，可以用CSS实现基础效果：**

#### 背景效果
```scss
// 浅色模式背景
.bg-light {
  background: linear-gradient(
    180deg,
    #FFF8EA 0%,
    #F5E6C3 50%,
    #F8B862 100%
  );
}

// 深色模式背景
.bg-dark {
  background: linear-gradient(
    180deg,
    #070B1F 0%,
    #1A1A3E 50%,
    #4A3F6B 100%
  );
}
```

#### 图标效果
```scss
// 用CSS实现简单图标
.icon-record {
  width: 24px;
  height: 24px;
  background: currentColor;
  mask: url('./record.svg') no-repeat center;
  mask-size: contain;
}
```

#### 装饰效果
```scss
// 用CSS实现星星
.star {
  width: 24px;
  height: 24px;
  background: #FFD700;
  clip-path: polygon(50% 0%, 61% 35%, 98% 35%, 68% 57%, 79% 91%, 50% 70%, 21% 91%, 32% 57%, 2% 35%, 39% 35%);
}
```

## 4. 素材制作建议

### 4.1 设计工具推荐

**首选：Figma**
- 免费版够用
- 协作方便
- 切图方便
- 有丰富插件

**备选：Sketch**
- Mac用户可选
- 插件丰富
- 性能好

### 4.2 设计规范

**颜色规范**
- 浅色模式主色：`#D99A28`（暖黄色）
- 深色模式主色：`#9273E6`（月光紫）
- 背景色：见设计系统文档

**尺寸规范**
- 图标：48x48px 或 32x32px（2倍图）
- 背景：750x1334px
- 装饰：多种尺寸

**风格规范**
- 圆角：大圆角（24px+）
- 线条：2px线宽
- 颜色：温暖、治愈
- 感觉：安静、私密

### 4.3 文件命名规范

```
[类型]-[名称]-[状态].[格式]

示例：
tab-record.png
tab-record-active.png
bg-light-sunset.png
deco-star-1.png
```

## 5. 协作流程

### 5.1 设计师工作流

1. **理解需求**：阅读需求方案和设计系统
2. **参考视觉稿**：基于现有视觉参考设计
3. **制作素材**：按规范制作图标和背景
4. **切图输出**：导出PNG文件
5. **标注文档**：提供颜色、尺寸标注

### 5.2 开发者工作流

1. **建立设计系统**：定义颜色、字体、间距变量
2. **实现基础组件**：用CSS实现卡片、按钮等
3. **搭建页面框架**：实现页面布局
4. **替换素材**：将CSS效果替换为图片
5. **优化细节**：添加动画、适配优化

### 5.3 协作要点

**设计师需要提供**
- Figma/Sketch源文件
- 切图文件（PNG格式）
- 颜色规范（HEX值）
- 尺寸规范（px值）

**开发者需要提供**
- 组件需求说明
- 使用场景描述
- 适配要求
- 性能要求
