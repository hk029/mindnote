# 「念头」小程序视觉设计规范

## 1. 产品概述

**一句话定义**
「念头」是一个记录起心动念的小程序 —— 一个安静放下念头、看见自己的地方。

**核心链路**
记下这一念 → 温柔倾听 → 整理成卡 → 回看发现

**产品定位边界：**
- 不是日记 App
- 不是情绪打卡工具
- 不是心理咨询工具
- 不需要长篇记录，不需要数据分析，不需要打卡激励

## 2. 视觉关键词与整体调性

| 关键词 | 含义 |
|--------|------|
| 温暖 | 不过度，刚好能接纳情绪的温度 |
| 安静 | 减少视觉噪音，让用户把注意力放回内心 |
| 治愈 | 被理解、被接住的感觉，而非刻意安慰 |
| 私密 | 像自己的小空间，不被打扰 |
| 克制 | 少即是多，不做多余装饰 |
| 有呼吸感 | 留白充足，节奏缓慢 |
| 不评判 | 任何念头都可以在这里安放 |

整体不要太工具化，也不要太心理咨询化。用户打开时应该感觉：这里可以慢一点，可以把心里的东西轻轻放下。

## 3. 主视觉方向

### 3.1 浅色模式 — 温暖治愈风
**关键词：** 米白、暖黄、奶油色、日落、海面、暖光、星星、小云朵、轻微手绘感

**感觉：** 念头被轻轻放在阳光里。

### 3.2 深色模式 — 深色静谧风
**关键词：** 深蓝黑、月光紫、夜空、星星、月亮、海上生明月、静水、远山

**感觉：** 在安静里，看见自己。

**模式对比**

| 维度 | 浅色模式 | 深色模式 |
|------|----------|----------|
| 主色调 | 米白 / 暖黄 / 奶油色 | 深蓝黑 / 月光紫 |
| 氛围意象 | 日落、海面、暖光 | 夜空、月亮、海上生明月、静水、远山 |
| 光感 | 暖光、光晕 | 月光、星光 |
| 手绘感 | 轻微手绘感 | 静谧细腻 |
| 情绪传达 | 温暖、被阳光拥抱 | 沉静、自我觉察 |

## 4. 核心视觉元素

| 元素 | 视觉意象 | 象征含义 |
|------|----------|----------|
| 海面 | 日落海面 / 海上生明月 | 念头流动 |
| 日落 | 暖橙色渐变 | 温暖 |
| 月亮 | 夜间静月 | 觉察 |
| 星星 | 散落星光 | 细微念头的闪光 |
| 小云朵 | 轻柔漂浮 | 轻盈、柔软 |
| 叶子 | 自然点缀 | 生命力、安静生长 |
| 光晕 | 柔和漫射光 | 温暖氛围 |
| 水面倒影 | 镜面般的水面 | 映照自己 |

首页可重点使用「日落海面」或「海上生明月」作为背景意象。

## 5. 页面约定

### 5.1 首页
**定位：** 让用户快速记录念头。

**核心内容：**
- 标题：「此刻，心里冒出了什么念头？」
- 输入框
- 语音按钮
- 主按钮：「记下这一念」
- 背景插画（日落海面 / 海上生明月）
- 底部导航

首页要干净，不要放太多入口。

### 5.2 温柔倾听页
**定位：** 对话式引导，不是客服聊天也不是问卷。感觉有人在安静地听我说话。

**视觉规范：**
- 对话气泡柔和，不要生硬直角
- 系统头像可用星星 / 月亮 / 云朵图标
- 用户气泡与系统气泡区分清楚（颜色 / 位置 / 对齐方向）
- 气泡之间留足呼吸感
- 底部输入框轻量

### 5.3 念头卡页
**定位：** 把一次念头整理成可回看的卡片，像一张温柔的「心念标本」。

**字段结构：**
- 原始念头（重点高亮）
- 当时
- 感受
- 看见
- 模式
- 提醒

**视觉规范：**
- 分区清晰，每个字段独立可辨
- 标签轻量，不喧宾夺主
- 不要做成长报告
- 整体呈现温柔、克制的标本感

### 5.4 心念本页
**定位：** 管理和找回用户保存过的念头。

**核心内容：**
- 标题：「心念本」
- 副标题：「收着你曾经认真看见过的念头」
- 最近 7 天摘要
- 筛选选项（全部 / 按感受 / 按模式 / 按原话）
- 记录列表

**视觉规范：**
- 列表卡片样式，默认展示
- 每条卡片展示：原始念头、简短背景、情绪标签、模式标签、一句提醒、时间
- 不要过度手账化，保证可读性

### 5.5 这一念详情页
**定位：** 单条念头的「心念标本」，用于凝视。

**视觉规范：**
- 可以更有纸张感、标本感
- 原始念头像被夹在纸页上的一句话
- 每个字段像标本标签
- 「提醒」像写给未来自己的小纸条

### 5.6 发现页
**定位：** 温柔回看最近的自己。

**内容模块：**
- 最近 7 天记录数量
- 最近常出现的念头
- 最近常见的模式
- 最近对自己说的话
- 念头墙

**视觉规范：**
- 使用卡片分组呈现
- 可以有轻量进度条
- 不要复杂图表
- 不要做成数据后台
- 卡片墙样式很适合发现页

### 5.7 我的页面
**定位：** 设置、隐私、本地数据管理。

**核心内容：**
- 主题模式切换
- 本地数据管理
- 隐私说明
- 清除全部记录
- 数据导出（后续）
- 反馈入口
- 关于产品

## 6. 统一约定

浅色模式和深色模式可以有不同氛围，但结构必须一致。

| 类别 | 可以变化 | 不能变化 |
|------|----------|----------|
| 配色方案 | ✅ | — |
| 背景插画 | ✅ | — |
| 图标颜色 | ✅ | — |
| 装饰元素 | ✅（如星星/光晕密度） | — |
| 光感氛围 | ✅ | — |
| 页面布局 | — | ❌ |
| 卡片结构 | — | ❌ |
| 信息层级 | — | ❌ |
| 按钮位置 | — | ❌ |
| 字段顺序 | — | ❌ |

尤其是「念头卡」和「发现页」，必须使用统一卡片系统。

## 7. 卡片系统规范

卡片是「念头」的核心视觉语言，承载以下内容类型：
- 输入框
- 对话气泡
- 念头卡
- 心念本列表
- 发现页统计
- 标签
- 念头墙

**卡片设计规则**

| 属性 | 浅色模式 | 深色模式 |
|------|----------|----------|
| 圆角 | 大圆角 | 大圆角 |
| 留白 | 舒服留白 | 舒服留白 |
| 阴影 / 描边 | 轻阴影或弱描边 | 轻阴影或弱描边 |
| 质感 | 偏纸张感 | 半透明玻璃感 |
| 密度 | 不要拥挤 | 不要拥挤 |

## 8. 图标风格

**方向：** 简洁、圆润、线面结合、不尖锐、不复杂、有陪伴感但不幼稚。

**底部导航图标参考**

| 导航项 | 图标方向 |
|--------|----------|
| 记录 | 叶子 / 笔 / 小纸片 |
| 心念本 | 书本 / 纸页 / 小花 |
| 发现 | 星星 / 罗盘 / 月亮 |
| 我的 | 人形 / 小屋 / 圆形头像 |

## 9. 避免方向

以下方向应严格避免：
- ❌ 医疗健康 App
- ❌ 严肃心理咨询 App
- ❌ 情绪打卡工具
- ❌ 数据分析后台
- ❌ 过度可爱儿童风
- ❌ 复杂冥想风
- ❌ 信息流社区风
- ❌ 过度国风禅意

美感来自：少、静、轻、暖。
